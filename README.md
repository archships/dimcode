# DimCode Issues

这里是 DimCode 的公开反馈与问题追踪仓库。

如果你在使用 DimCode 时遇到问题、发现体验不顺、希望提出功能建议，欢迎通过本仓库提交 Issue。我们会在这里收集反馈、澄清需求、同步处理进展，并将有价值的问题纳入后续版本规划。

## 可以提交什么

欢迎提交以下类型的反馈：

- **Bug Report**：功能异常、崩溃、卡顿、显示错误、结果不符合预期。
- **Feature Request**：希望新增的能力、工作流改进或集成建议。
- **UX Feedback**：界面、交互、文案、设置流程等使用体验问题。
- **Compatibility Issue**：不同系统、设备、模型服务或项目环境下的兼容性问题。
- **Question**：使用过程中不确定的行为、配置或限制说明。

## 提交前请先检查

提交 Issue 前，建议先做一次简单检查：

- 搜索是否已有相同或相似 Issue。
- 确认你使用的是较新的 DimCode 版本。
- 尽量准备可复现步骤、截图、录屏或错误信息。
- 如果问题涉及第三方服务，请说明服务名称和大致配置方式，但不要提交密钥。


## 安全与隐私

请不要在公开 Issue 中提交任何敏感信息，包括但不限于：

- API Key、Token、密码、私钥
- 个人身份信息、邮箱、手机号
- 公司内部代码、客户数据、未公开文档
- 完整日志中可能包含的本地路径、环境变量或请求内容

如果你需要提供日志，请先手动脱敏，只保留与问题相关的片段。

## 处理流程

Issue 提交后，我们通常会：

1. 确认问题类型和影响范围。
2. 在需要时向你补充询问信息。
3. 根据优先级安排修复、优化或产品评估。
4. 在 Issue 中同步处理状态或关闭原因。

我们会尽量保持沟通清晰，但无法保证所有建议都会立即进入开发计划。

## 关于 DimCode

DimCode 是一个面向开发者的 AI 编程工具，目标是帮助用户更自然地在本地项目中进行对话、理解上下文、处理代码任务和管理工作流。

本仓库仅用于公开反馈与问题追踪，不包含 DimCode 应用源码。
